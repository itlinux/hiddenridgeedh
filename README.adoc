== Features

[cols="1,3"]
|===
| Feature | Description

| Blog / News | Admin-authored posts with rich text, categories, tags, cover images
| Photo Gallery | Member-uploaded photos with auto-thumbnails, tags, lightbox
| Events Calendar | Create events, RSVP, max attendees, status tracking
| Forum | Threaded discussions, pinning, locking
| Member Directory | Approved neighbor profiles (login required)
| Newsletter | SendGrid integration, per-subscriber unsubscribe tokens
| Safety & Alerts | Neighborhood alerts via web or SMS (Twilio), emergency contacts, safety tips
| SMS-to-Alert | Text your Twilio number to auto-post alerts (registered members only)
| Neighborhood Map | Leaflet/OpenStreetMap map of member locations (login required)
| Two-Factor Auth | TOTP-based 2FA with QR code setup and backup codes
| Role-based Access | Super Admin → Content Admin → Approved Member → Pending
|===

== Roles

[cols="1,3"]
|===
| Role | Can Do

| `super_admin` | Everything — approve members, manage roles, send newsletters, all CRUD
| `content_admin` | Create/edit/delete posts, events, gallery; moderate forum; lock threads
| `member` | Read all content, create forum threads/replies, upload gallery photos, RSVP
| `pending` | Registered but awaiting approval — cannot access member content
|===

== Tech Stack

* *Frontend:* Next.js 14 (App Router), TypeScript, Tailwind CSS
* *Backend:* FastAPI (Python 3.13), Motor (async MongoDB driver)
* *Database:* MongoDB (system instance on port 27017, db: `hiddenridgeedh`)
* *Email:* SendGrid (SMTP support planned)
* *SMS:* Twilio (inbound webhook for SMS-to-alert)
* *Images:* Pillow (auto-thumbnail generation), served via Nginx
* *Auth:* JWT (python-jose), bcrypt, TOTP 2FA (pyotp)
* *Security:* slowapi rate limiting, Turnstile CAPTCHA, nginx hardening
* *Server:* Rocky Linux, Nginx, systemd, Let's Encrypt TLS

== Setup

=== 1. Environment Variables

[source,bash]
----
cp backend/.env.example backend/.env
# Edit backend/.env with your values
----

Key variables:

* `SECRET_KEY` — generate with `openssl rand -hex 32`
* `MONGODB_URL` — default `mongodb://localhost:27017`
* `MONGODB_DB` — default `hiddenridgeedh`
* `SENDGRID_API_KEY` — from SendGrid dashboard
* `ADMIN_EMAIL` — your email for admin alerts
* `BOOTSTRAP_SECRET` — secret for initial admin bootstrap
* `TURNSTILE_SECRET_KEY` — Cloudflare Turnstile (optional, skipped if empty)
* `TWILIO_ACCOUNT_SID` — Twilio account SID (optional, for SMS alerts)
* `TWILIO_AUTH_TOKEN` — Twilio auth token
* `TWILIO_PHONE_NUMBER` — your Twilio phone number (e.g. +19165551234)

=== 2. Development

*Backend:*

[source,bash]
----
cd backend
python -m venv ../venv
source ../venv/bin/activate
pip install -r requirements.txt
uvicorn main:app --host 127.0.0.1 --port 8003 --reload
----

*Frontend:*

[source,bash]
----
cd frontend
npm install
echo "NEXT_PUBLIC_API_URL=http://localhost:8003" > .env.local
npm run dev
----

* Frontend: http://localhost:3000
* Backend API: http://localhost:8003
* API Docs: http://localhost:8003/docs

=== 3. Bootstrap Super Admin

After first start, create the super admin account:

[source,bash]
----
curl -X POST "http://localhost:8003/api/edh/bootstrap?secret=YOUR_BOOTSTRAP_SECRET"
----

Then log in and change the default password.

== SMS Alerts (Twilio)

Members with SMS opt-in can text the neighborhood Twilio number to post alerts directly to the Safety page.

=== How It Works

1. Member enables SMS opt-in and sets their phone number on the profile page
2. Member texts a message to the Twilio phone number
3. Twilio POSTs to `POST /api/alerts/sms/inbound` with sender phone and message body
4. Backend matches the phone to a registered user (last 10 digits)
5. Alert is created and appears on the `/safety` page with an "SMS" badge
6. Twilio replies with a confirmation message (requires A2P 10DLC registration)

=== Setup

1. Create a Twilio account and buy a phone number
2. Add to `backend/.env`:
+
[source]
----
TWILIO_ACCOUNT_SID=your_account_sid
TWILIO_AUTH_TOKEN=your_auth_token
TWILIO_PHONE_NUMBER=+19165551234
----
3. In Twilio Console -> Phone Numbers -> your number -> Messaging:
   * "A message comes in" -> Webhook -> `https://hiddenridgeedh.com/api/alerts/sms/inbound` (HTTP POST)
4. Complete A2P 10DLC registration for outbound replies (optional but recommended)

=== Cost

Twilio local number: ~$1.15/month + $0.0079/inbound SMS + $0.0079/outbound SMS.

== Security

=== Nginx Hardening

* Blocks hidden files, WordPress/PHP attack paths, scanner bots
* Blocks suspicious file extensions (`.php`, `.asp`, `.exe`, etc.)
* Blocks known scanner user-agents (sqlmap, nikto, nmap, etc.)
* Security headers: HSTS, CSP, X-Frame-Options, X-Content-Type-Options, Referrer-Policy, Permissions-Policy

=== Application Security

* *Rate limiting* (slowapi): 120 req/min default, stricter on auth/newsletter/alerts
* *Login brute-force protection*: IP-based blocking after 5 failed attempts (15-min lockout), with CIDR whitelist
* *Input validation*: Pydantic Field constraints (min/max length) on all user inputs
* *Password policy*: 8+ chars, uppercase, lowercase, number required
* *Turnstile CAPTCHA*: Cloudflare Turnstile on register/login forms
* *TOTP 2FA*: Optional two-factor authentication with backup codes
* *Twilio signature validation*: HMAC-SHA1 verification on SMS webhook
